云计算技术 课程实验报告：个人博客系统搭建

课程名称 云计算技术 实验项目 个人博客系统的搭建与云端部署
实验环境 Windows 实验日期 2026.3.20

一、 实验目的

本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。
具体包括：
1.构建环境并实现个人博客
2.网站的搭建，撰写实验报告，并将这份报告作为一篇博文发布在自己的博客 / 网站上。

二、 实验环境

  • 硬件：联网计算机一台
  • 软件：
    • Git
    • Node.js (Hexo 运行环境)
    • Hexo CLI (博客生成工具)
    • VS Code / Notepad++

三、 实验步骤

1. 本地环境搭建

首先，在本地计算机安装基础环境：

  • 安装 Node.js：从官网下载并配置环境变量，确保 npm 命令可用。
  • 安装 Git：用于与 GitHub 服务器通信。
  • 安装 Hexo：执行命令 npm install -g hexo-cli。

2. GitHub 仓库准备（云端环境）

这是实现博客公网访问的关键步骤：

  1. 登录 GitHub：访问 github.com。
  2. 新建仓库：点击 “New repository”。
  3. 命名规范：仓库名必须为 用户名.github.io（例如：myname.github.io）。
  4. 权限设置：选择 Public，并勾选 Add a README file。

3. 配置 SSH 免密登录

为了让本地计算机有权限将代码推送到 GitHub，需要建立信任关系：

  1. 生成密钥：在终端输入 ssh-keygen -t rsa -C "your_email@example.com"。
  2. 获取公钥：复制 ~/.ssh/id_rsa.pub 文件中的内容。
  3. 绑定 GitHub：在 GitHub 的 Settings -> SSH and GPG keys 中添加新的 SSH Key。
  4. 验证连接：输入 ssh -T git@github.com，看到 “Hi [username]!” 表示成功。

四、 实验结果展示

1. 部署过程日志

执行 hexo d 后，终端输出 Deploy done: git，表示文件已成功上传至 GitHub Pages 所在的云计算节点。

2. 网站访问效果

在浏览器输入 https://您的用户名.github.io，即可查看到搭建完成的个人博客。

3. 实验报告发布

将本 Markdown 文件放入 source/_posts/ 文件夹，再次执行 hexo g -d，本实验报告即在公网环境下实时呈现。

五、 实验总结

1. 遇到的问题与解决方案

  • 问题：执行 hexo d 时提示 Permission denied (publickey)。
    • 解决：发现是由于 SSH Key 没有添加到 GitHub 的设置中，重新生成并绑定公钥后解决。
  • 问题：部署后网页样式混乱（CSS失效）。
    • 解决：修改 _config.yml 中的 url 和 root 配置，确保其指向正确的 GitHub 域名。

2. 实验心得

通过本次实验，我深刻理解了“软件即服务”（SaaS）和“平台即服务”（PaaS）在实际开发中的应用。GitHub Pages 提供了一个免费且稳定的静态网站托管环境，省去了购买服务器和配置 Nginx 的繁琐步骤。
同时，我也掌握了 Git 工作流和 Hexo 框架的配合使用。这种“本地撰写 -> 云端部署”的模式极大提高了文档发布的效率，为后续深入学习云计算技术打下了良好的基础。
